# 08. 業務フロー図

## カラー凡例

| 色 | 意味 |
|----|------|
| 薄サーモン | 課題・非効率が発生している箇所（AS-IS） |
| 薄緑 | 合格・完了・成功 |
| 薄黄 | 警告・注意・保留 |
| 薄青 | システム処理・情報 |
| 薄赤 | 不合格・廃棄・キャンセル |

## 8.1 全体業務フロー（AS-IS：現状）

```mermaid
flowchart TD
    A([受注]) --> B["生産計画立案<br/>紙・Excel"]
    B --> C["製造指示発行<br/>紙・Excel印刷"]
    C --> D[製造現場へ紙配布]
    D --> E["原材料払出<br/>現場確認・手書き台帳"]
    E --> F["製造工程<br/>加工・組立"]
    F --> G["工程内検査<br/>紙記録"]
    G --> H{合否判定}
    H -- 不合格 --> I["不良対応<br/>紙で記録・口頭連絡"]
    I --> F
    H -- 合格 --> J["完成品検査<br/>紙記録"]
    J --> K{最終判定}
    K -- 不合格 --> I
    K -- 合格 --> L["完成品入庫<br/>手書き台帳"]
    L --> M["出荷指示<br/>紙・Excel"]
    M --> N[ピッキング・梱包]
    N --> O([出荷・顧客納品])

    style B fill:#ffd2cc,stroke:#c0392b,color:#7b241c
```

## 8.2 全体業務フロー（TO-BE：新システム導入後）

```mermaid
flowchart TD
    subgraph EXT[外部システム]
        A([受注システム])
    end

    subgraph SYS[製造管理システム]
        B[生産計画管理]
        C[製造指示管理]
        D["在庫管理<br/>原材料払出"]
        E["工程管理<br/>実績入力"]
        F["品質管理<br/>検査入力"]
        G["在庫管理<br/>完成品入庫"]
        H[出荷管理]
    end

    subgraph ERP[ERP / 会計システム]
        I([実績・在庫連携])
    end

    A -- 受注情報連携 --> B
    B -- 計画確定・一括展開 --> C
    C -- 原材料引当指示 --> D
    D -- 払出完了 --> E
    E -- 工程完了実績 --> F
    F -- 合否判定 --> J{合否}
    J -- 不合格 --> K["不良登録・処置"]
    K --> E
    J -- 合格 --> G
    G -- 完成品入庫 --> H
    H --> L([顧客納品])
    G -- 在庫実績 --> I
    E -- 製造実績 --> I

    style SYS fill:#dbeafe,stroke:#1d4ed8
    style EXT fill:#f3f4f6,stroke:#6b7280
    style ERP fill:#f3f4f6,stroke:#6b7280
    style K fill:#fadbd8,stroke:#c0392b,color:#922b21
```

## 8.3 製造指示管理フロー

```mermaid
flowchart TD
    A([生産計画 確定]) --> B["製造指示 一括展開<br/>/手動作成"]
    B --> C{在庫引当<br/>確認}
    C -- "在庫不足<br/>警告あり" --> D["警告表示<br/>※発行は可能"]
    C -- 在庫OK --> E[製造指示 発行]
    D --> E
    E --> F["製造指示番号 採番<br/>MO-YYYYMMDD-NNNN"]
    F --> G["現場端末へ通知<br/>タブレット配信"]

    G --> H{現場担当者<br/>確認}
    H -- 着手 --> I["工程着手登録<br/>着手中ステータス"]
    I --> J[製造作業]
    J --> K["工程完了登録<br/>実績数量入力"]
    K --> L{全工程<br/>完了？}
    L -- No --> J
    L -- Yes --> M["製造指示 完了<br/>完成品入庫へ"]

    subgraph 変更・キャンセル
        N["製造指示 変更<br/>未着手のみ数量変更可"]
        O["製造指示 キャンセル<br/>未着手・着手中のみ"]
    end

    E -.-> N
    E -.-> O

    style D fill:#fef3c7,stroke:#d97706,color:#78350f
    style E fill:#d5f5e3,stroke:#1e8449,color:#1e8449
    style M fill:#d5f5e3,stroke:#1e8449,color:#1e8449
    style N fill:#dbeafe,stroke:#1d4ed8,color:#1e3a8a
    style O fill:#fadbd8,stroke:#c0392b,color:#922b21
```

## 8.4 品質管理フロー

```mermaid
flowchart TD
    subgraph 受入検査
        A([原材料入荷]) --> B[受入検査 実施]
        B --> C{受入判定}
        C -- 合格 --> D[利用可能在庫へ]
        C -- 不合格 --> E["不良登録<br/>返品 / 廃棄 / 特採"]
    end

    subgraph 工程内検査
        F([工程着手]) --> G[工程内検査 実施]
        G --> H{工程内判定}
        H -- 合格 --> I[次工程へ]
        H -- 不合格 --> J["不良登録<br/>手直し / 廃棄"]
        J --> K{手直し<br/>可否}
        K -- 手直し --> G
        K -- 廃棄 --> L[廃棄処置]
    end

    subgraph 完成品検査
        M([全工程完了]) --> N[完成品検査 実施]
        N --> O{最終判定}
        O -- 合格 --> P["完成品入庫<br/>出荷可能"]
        O -- 不合格 --> Q["不良登録<br/>処置決定"]
        Q --> R{処置区分}
        R -- 手直し --> N
        R -- 特採 --> P
        R -- 廃棄 --> S[廃棄処置]
    end

    D --> F
    I --> F
    P --> T([出荷管理へ])

    style D fill:#d5f5e3,stroke:#1e8449,color:#1e8449
    style I fill:#d5f5e3,stroke:#1e8449,color:#1e8449
    style P fill:#d5f5e3,stroke:#1e8449,color:#1e8449
    style E fill:#fadbd8,stroke:#c0392b,color:#922b21
    style L fill:#fadbd8,stroke:#c0392b,color:#922b21
    style S fill:#fadbd8,stroke:#c0392b,color:#922b21
```

## 8.5 在庫管理フロー（入出庫）

```mermaid
flowchart TD
    subgraph 入庫
        A1([購買入荷]) --> B1[受入検査]
        B1 --> C1{合否}
        C1 -- 合格 --> D1["入庫登録<br/>ロット・棚番指定"]
        C1 -- 不合格 --> E1["検査中在庫 / 返品"]
        A2([製造完成]) --> D1
        D1 --> F1["在庫加算<br/>利用可能在庫"]
    end

    subgraph 在庫管理
        F1 --> G1[在庫照会]
        G1 --> H1{安全在庫<br/>以下？}
        H1 -- Yes --> I1["アラート通知<br/>調達部門へ"]
        H1 -- No --> G1
    end

    subgraph 出庫
        J1([製造指示 引当]) --> K1["在庫引当<br/>FIFO"]
        K1 --> L1["払出登録<br/>バーコードスキャン"]
        L1 --> M1[在庫減算]
        N1([出荷指示]) --> L1
    end

    subgraph 棚卸
        O1["棚卸開始宣言<br/>在庫ロック"] --> P1["実地数量入力<br/>ハンディ端末"]
        P1 --> Q1["差異確認<br/>帳簿 vs 実地"]
        Q1 --> R1{差異あり？}
        R1 -- Yes --> S1["差異調整入力<br/>承認"]
        R1 -- No --> T1[棚卸完了]
        S1 --> T1
    end

    style D1 fill:#d5f5e3,stroke:#1e8449,color:#1e8449
    style F1 fill:#d5f5e3,stroke:#1e8449,color:#1e8449
    style E1 fill:#fadbd8,stroke:#c0392b,color:#922b21
    style I1 fill:#fef3c7,stroke:#d97706,color:#78350f
    style M1 fill:#dbeafe,stroke:#1d4ed8,color:#1e3a8a
    style T1 fill:#d5f5e3,stroke:#1e8449,color:#1e8449
```

## 8.6 出荷管理フロー

```mermaid
flowchart TD
    A(["受注情報<br/>外部システム連携"]) --> B[出荷指示 登録]
    B --> C{完成品検査<br/>合格済み？}
    C -- "未検査/不合格" --> D["出荷保留<br/>品質部門へ連絡"]
    D --> C
    C -- 合格済み --> E[出荷指示 確定]
    E --> F[ピッキングリスト 出力]
    F --> G["ピッキング作業<br/>バーコードスキャン確認"]
    G --> H{数量<br/>一致確認}
    H -- 不一致 --> I[数量確認・修正]
    I --> H
    H -- 一致 --> J[梱包作業]
    J --> K["出荷実績 登録<br/>実出荷日時・ロット確定"]
    K --> L["在庫減算<br/>完成品出庫"]
    L --> M["納品書・送り状 発行<br/>PDF出力"]
    M --> N["出荷完了通知<br/>受注システムへ連携"]
    N --> O([顧客納品])

    P([顧客からの返品]) --> Q[返品受付登録]
    Q --> R[品質確認]
    R --> S{再利用<br/>可否}
    S -- 可 --> T["返品入庫<br/>在庫加算"]
    S -- 不可 --> U[廃棄処置]

    style D fill:#fef3c7,stroke:#d97706,color:#78350f
    style E fill:#d5f5e3,stroke:#1e8449,color:#1e8449
    style O fill:#d5f5e3,stroke:#1e8449,color:#1e8449
    style T fill:#dbeafe,stroke:#1d4ed8,color:#1e3a8a
    style U fill:#fadbd8,stroke:#c0392b,color:#922b21
```

## 8.7 トレーサビリティ追跡フロー

```mermaid
flowchart LR
    subgraph FWD[フォワードトレース：原材料 → 顧客]
        direction LR
        A[原材料ロット] --> B[受入検査]
        B --> C[在庫引当]
        C --> D[製造指示]
        D --> E[製造工程]
        E --> F[品質検査]
        F --> G[完成品ロット]
        G --> H[出荷]
        H --> I[顧客・出荷先]
    end

    subgraph BWD[バックワードトレース：完成品 → 仕入先]
        direction RL
        I2["完成品ロット<br/>/シリアル"] --> J[出荷情報]
        J --> K[検査実績]
        K --> L[製造指示]
        L --> M[使用原材料ロット]
        M --> N[仕入先情報]
    end

    style FWD fill:#dbeafe,stroke:#1d4ed8
    style BWD fill:#fef3c7,stroke:#d97706
```
