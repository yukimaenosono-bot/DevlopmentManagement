# 04. 非機能要件

## 4.1 性能要件

| # | 要件 | 目標値 | 計測条件 |
|---|------|--------|---------|
| P-001 | 画面応答時間（通常操作） | 3秒以内 | 同時接続ユーザー数：通常時 |
| P-002 | 画面応答時間（検索・集計） | 10秒以内 | 1年分のデータ、同時接続ユーザー数：通常時 |
| P-003 | 一括処理（バッチ）完了時間 | - | - |
| P-004 | 同時接続ユーザー数 | - 名以上 | ピーク時 |
| P-005 | データ保持期間 | - 年以上 | 各業務トランザクションデータ |

## 4.2 可用性・信頼性要件

| # | 要件 | 目標値 | 備考 |
|---|------|--------|------|
| A-001 | システム稼働率 | 99.5%以上/月 | 計画停止を除く |
| A-002 | 計画停止時間 | 月 - 時間以内 | メンテナンス時間 |
| A-003 | 障害復旧時間（RTO） | - 時間以内 | - |
| A-004 | 目標復旧時点（RPO） | - 時間前 | データバックアップ基準 |
| A-005 | バックアップ頻度 | 日次 | フルバックアップ |

## 4.3 セキュリティ要件

| # | 要件 | 内容 |
|---|------|------|
| S-001 | 認証 | ユーザーID・パスワードによる認証（多要素認証は別途検討） |
| S-002 | アクセス制御 | ロールベースアクセス制御（RBAC）。機能・データへのアクセスを役割単位で制御 |
| S-003 | 通信暗号化 | HTTPS（TLS 1.2以上）による暗号化通信 |
| S-004 | セッション管理 | 一定時間操作がない場合の自動タイムアウト（- 分） |
| S-005 | ログ管理 | 操作ログ・ログイン履歴の記録（改ざん不可） |
| S-006 | パスワードポリシー | - 文字以上、英数字記号混在、- 日ごとの変更推奨 |
| S-007 | 不正アクセス対策 | 一定回数ログイン失敗でアカウントロック |

## 4.4 拡張性・保守性要件

| # | 要件 | 内容 |
|---|------|------|
| M-001 | スケールアウト対応 | ユーザー数・データ量増加に対応できるアーキテクチャ |
| M-002 | モジュール分離 | 機能ごとにモジュール化し、独立して変更・デプロイ可能 |
| M-003 | ログ出力 | システムエラー・例外は全てログ出力 |
| M-004 | 監視 | サーバリソース・アプリ死活監視の実装 |
| M-005 | ドキュメント整備 | API仕様書・DB定義書・運用手順書の整備 |

## 4.5 ユーザビリティ要件

| # | 要件 | 内容 |
|---|------|------|
| U-001 | 現場端末対応 | タブレット（Android/iPad）・ハンディターミナルのUI対応 |
| U-002 | バーコード・QRスキャン | 主要入力をスキャン操作で完結できること |
| U-003 | 多言語対応 | 日本語。将来的な多言語対応を考慮した設計（必要に応じて） |
| U-004 | オフライン対応 | ネットワーク断時の最低限操作継続（要検討） |

## 4.6 法規制・コンプライアンス要件

| # | 要件 | 内容 |
|---|------|------|
| C-001 | 個人情報保護 | 個人情報を含む場合、個人情報保護法に準拠した管理 |
| C-002 | トレーサビリティ記録保持 | 業界規制に応じたトレーサビリティ記録の保持期間を設定 |
| C-003 | 監査対応 | 操作・変更履歴の保持（ISO9001等の品質監査対応） |

## 4.7 移行要件

| # | 要件 | 内容 |
|---|------|------|
| T-001 | マスタデータ移行 | 品目・工程・設備等のマスタデータを現行システムから移行 |
| T-002 | トランザクションデータ移行 | 在庫残高等の初期データを移行する（移行範囲は別途確定） |
| T-003 | 並行稼働期間 | 本番稼働後 - ヶ月間の並行稼働を実施（要検討） |
