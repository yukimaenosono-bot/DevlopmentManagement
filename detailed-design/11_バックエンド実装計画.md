# 11. バックエンド実装計画

## 概要

本ドキュメントは Synapse バックエンド（.NET 10 / Clean Architecture / CQRS）の全実装計画を定義する。
既存実装の棚卸し→即時修正→未実装ドメインの段階的実装という順序で進める。

---

## Phase 0: 即時修正 — OpenAPI レスポンス型の修正

### 目的

フロントエンドの API-First フロー（`pnpm generate` で型を自動生成）を機能させる。
現状は全エンドポイントが `void` として生成されるため、手動で `types.ts` を維持している。

### 修正方針

すべての Controller に `[ProducesResponseType]` を追加し、DTO を返すよう修正する。

### 対象ファイルと修正内容

#### `WorkOrdersController.cs`

```csharp
[HttpGet]
[ProducesResponseType(typeof(IEnumerable<WorkOrderDto>), StatusCodes.Status200OK)]
public async Task<ActionResult<IEnumerable<WorkOrderDto>>> GetList(...)

[HttpGet("{id:guid}")]
[ProducesResponseType(typeof(WorkOrderDto), StatusCodes.Status200OK)]
[ProducesResponseType(StatusCodes.Status404NotFound)]
public async Task<ActionResult<WorkOrderDto>> GetById(...)

[HttpPost]
[ProducesResponseType(typeof(object), StatusCodes.Status201Created)]  // { id: Guid }
[ProducesResponseType(StatusCodes.Status400BadRequest)]
public async Task<ActionResult> Create(...)

// 状態遷移（cancel/start/complete）: 204 NoContent のまま（変更なし）
```

#### `ItemsController.cs`

```csharp
[HttpGet]
[ProducesResponseType(typeof(IEnumerable<ItemDto>), StatusCodes.Status200OK)]
public async Task<ActionResult<IEnumerable<ItemDto>>> GetList(...)

[HttpGet("{id:guid}")]
[ProducesResponseType(typeof(ItemDto), StatusCodes.Status200OK)]
[ProducesResponseType(StatusCodes.Status404NotFound)]
public async Task<ActionResult<ItemDto>> GetById(...)
```

#### その他 Controller（Processes / Equipments / Warehouses / Bom / Inventory / Users）

同様のパターンで `[ProducesResponseType]` を追加。

### ビルド・生成手順

```bash
cd development/backend && dotnet build  # openapi.json 更新
cd development/frontend && pnpm generate  # 型・Hooks 再生成
```

## Phase 1: 工程ルーティング / 工程実績

### 背景と設計方針

製造指示に対して「どの工程を、どの順番で行うか」を定義するのがルーティング。
各工程の着手・完了・実績数量を記録するのが工程実績（ProcessOperation）。

### 1-1. 新規 Entity 設計

#### `Routing`（工程ルーティングマスタ）

品目に対する標準工程順序を定義するマスタ。

```csharp
// Synapse.Domain/Entities/Routing.cs
public class Routing : Entity
{
    public Guid ItemId { get; private set; }
    public Item Item { get; private set; } = null!;
    public string Name { get; private set; }          // ルーティング名
    public bool IsDefault { get; private set; }       // 品目のデフォルトルーティング
    public IReadOnlyList<RoutingStep> Steps => _steps.AsReadOnly();
    private List<RoutingStep> _steps = new();

    public static Routing Create(Guid itemId, string name, bool isDefault);
    public void AddStep(RoutingStep step);
    public void RemoveStep(int sequence);
}

public class RoutingStep : Entity
{
    public Guid RoutingId { get; private set; }
    public int Sequence { get; private set; }         // 工程順序（10, 20, 30...）
    public Guid ProcessId { get; private set; }
    public Process Process { get; private set; } = null!;
    public Guid? EquipmentId { get; private set; }   // 使用設備（任意）
    public decimal? StandardTime { get; private set; } // 標準作業時間（分）
    public bool IsRequired { get; private set; }      // 必須工程か
}
```

### 2-3. `ExpandToWorkOrdersCommand` の実装詳細

```
1. 計画が「確定」状態であることを確認
2. BOM を取得し、必要な部品在庫を確認（引当チェック）
3. WorkOrder を作成（ProductionPlanNumber を設定）
4. 計画ステータスを InProgress に変更
5. 展開した WorkOrder ID リストを返す
```


### 4-3. `RecordShipmentCommand` の実装詳細

```
1. ShipmentOrder が「ピッキング済」であることを確認
2. 各明細の在庫引当（Stock から IssueInventoryCommand 実行）
3. 在庫が不足する場合はエラー返却（ロールバック）
4. 明細の ActualQuantity / ActualShipAt / HandledBy を更新
5. ShipmentOrder ステータスを Shipped に変更
```

```csharp
// Synapse.Infrastructure/Jobs/DelayAlertJob.cs
public class DelayAlertJob
{
    public async Task CheckDelayedOperations()
    {
        var today = DateOnly.FromDateTime(DateTime.UtcNow);
        var delayed = await _db.WorkOrders
            .Where(wo =>
                wo.Status == WorkOrderStatus.InProgress &&
                wo.PlannedEndDate < today)
            .ToListAsync();

        foreach (var wo in delayed)
        {
            // 通知（SignalR / メール等）
            await _hub.Clients.Group("factory")
                .SendAsync("WorkOrderDelayed", new { wo.Id, wo.WorkOrderNumber });
        }
    }
}
```

```csharp

RecurringJob.AddOrUpdate<DelayAlertJob>(
    "delay-alert",

);
```
