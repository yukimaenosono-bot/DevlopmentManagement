## 1. 実装フェーズ計画

`generated/` に API クライアントが存在するドメインを優先する。

| フェーズ | ドメイン | generated/ | 優先度 |
|---|---|---|---|
| 1 | 製造指示（work-orders） | ✅ あり | 最高 |
| 2 | マスタ管理（items / processes / equipments / warehouses / bom） | ✅ あり | 高 |
| 3 | 在庫管理（inventory） | ✅ あり | 高 |
| 4 | ユーザー管理（users） | ✅ あり | 中 |
| 5 | 生産計画 / 品質管理 / 出荷 / 工程進捗 | ❌ 未生成 | バックエンド実装後 |

フェーズ5の画面は、バックエンド実装 → `dotnet build` → `pnpm generate` の黄金サイクル実行後に着手する。

## 2. API レスポンス型について

現時点で `generated/` の関数は `apiClient<void>` となっており、レスポンス型が `void` になっている（OpenAPI スキーマにレスポンスボディの型定義が未設定のため）。

**対応方針**: 各 `features/[domain]/hooks/` で型アサーションを行い、バックエンドの実際のレスポンス構造に合わせた型を一時的に補完する。OpenAPI のレスポンス型が正式に定義されたら `pnpm generate` で上書きする。

```typescript
// 暫定対応パターン（features/manufacturing-order/hooks/useWorkOrders.ts）
import { useGetApiWorkOrders } from '@/generated/work-orders'

// バックエンドのレスポンスに合わせた仮型（generated が void のため）
type WorkOrderListItem = {
  id: string
  workOrderNumber: string
  itemCode: string
  itemName: string
  quantity: number
  plannedStartDate: string
  plannedEndDate: string
  status: number  // WorkOrderStatus enum
  totalOperations: number
  completedOperations: number
}

export function useWorkOrders(params?: ...) {
  return useGetApiWorkOrders(params) as UseQueryResult<WorkOrderListItem[]>
}
```

## 3. 共通 enum 定数定義

`WorkOrderStatus` 等は `generated/model/` では `number` 型として生成される。
表示ラベル・カラー変換は `features/[domain]/constants.ts` に定義する。

```typescript
// features/manufacturing-order/constants.ts
export const WORK_ORDER_STATUS = {
  0: { label: '未着手', color: 'gray' },
  1: { label: '着手中', color: 'blue' },
  2: { label: '完了',   color: 'green' },
  3: { label: 'キャンセル', color: 'red' },
} as const
```

## 5. 画面別コンポーネント設計

### 凡例

- **Page**: `pages/` の薄いラッパー（import + return のみ）
- **Feature コンポーネント**: `features/[domain]/components/` に実装
- **共通コンポーネント**: `components/composite/` または `components/ui/` を使用

### 5-1. SCR-MO-001 製造指示一覧（`/manufacturing-orders`）

```
WorkOrderListPage（pages/manufacturing-order/WorkOrderListPage.tsx）
```

### 5-9. マスタ管理（フェーズ2）
