# 05. データフロー図

## システム全体のデータフロー

```mermaid
flowchart TD
    subgraph EXT_IN[外部システム（入力）]
        ORDER_SYS[受注システム]
        PURCHASE_SYS[調達システム]
    end

    subgraph EXT_OUT[外部システム（出力）]
        ERP[ERP / 会計システム]
        PURCHASE_SYS_OUT[調達システム]
    end

    subgraph CORE[SYNAPSE]
        PP_MODULE[生産計画管理]
        MO_MODULE[製造指示管理]
        PROC_MODULE[工程管理]
        INV_MODULE[在庫管理]
        QC_MODULE[品質管理]
        SH_MODULE[出荷管理]
    end

    subgraph DB[データ層]
        MASTER_DB[(マスタDB)]
        TRANS_DB[(トランザクションDB)]
        FILE_STORE["Cloud Storage<br/>帳票PDF"]
    end

    subgraph USERS[利用者]
        PROD_MGR[生産管理担当]
        FIELD_WORKER[現場作業員]
        QC_STAFF[品質管理担当]
        WAREHOUSE[倉庫担当]
        SHIPPING[出荷担当]
    end

    %% 外部連携
    ORDER_SYS -- 受注情報（Webhook） --> PP_MODULE
    PURCHASE_SYS -- 入荷情報（REST API） --> INV_MODULE
    INV_MODULE -- 在庫アラート（REST API） --> PURCHASE_SYS_OUT
    CORE -- 製造実績・在庫残高（CSV日次） --> ERP

    %% 生産計画 → 製造指示
    PROD_MGR --> PP_MODULE
    PP_MODULE -- 計画データ --> TRANS_DB
    PP_MODULE -- 製造指示展開 --> MO_MODULE
    MO_MODULE -- 製造指示データ --> TRANS_DB

    %% 製造指示 → 工程管理
    MO_MODULE -- SignalRプッシュ配信 --> FIELD_WORKER
    FIELD_WORKER -- 工程着手・完了 --> PROC_MODULE
    PROC_MODULE -- 工程実績 --> TRANS_DB

    %% 在庫管理
    WAREHOUSE -- 入出庫登録 --> INV_MODULE
    MO_MODULE -- 原材料引当 --> INV_MODULE
    INV_MODULE -- 在庫データ --> TRANS_DB

    %% 品質管理
    QC_STAFF -- 検査結果入力 --> QC_MODULE
    QC_MODULE -- 検査実績・不良情報 --> TRANS_DB
    QC_MODULE -- 合否フィードバック --> MO_MODULE

    %% 出荷管理
    SHIPPING -- 出荷実績登録 --> SH_MODULE
    SH_MODULE -- 出荷実績データ --> TRANS_DB
    SH_MODULE -- 帳票PDF生成 --> FILE_STORE
    SH_MODULE -- 在庫減算 --> INV_MODULE

    %% マスタ参照
    MASTER_DB --> PP_MODULE
    MASTER_DB --> MO_MODULE
    MASTER_DB --> PROC_MODULE
    MASTER_DB --> INV_MODULE
    MASTER_DB --> QC_MODULE
    MASTER_DB --> SH_MODULE
```

## 主要業務フロー別データフロー

### 製造指示〜工程完了

```mermaid
sequenceDiagram
    participant 受注SYS as 受注システム
    participant 生産計画 as 生産計画管理
    participant 製造指示 as 製造指示管理
    participant 在庫 as 在庫管理
    participant 現場端末 as 現場端末（タブレット）
    participant 工程管理 as 工程管理

    受注SYS->>生産計画: 受注情報（Webhook）
    生産計画->>製造指示: 製造指示展開
    製造指示->>在庫: 原材料引当確認
    在庫-->>製造指示: 引当可否
    製造指示->>現場端末: SignalRプッシュ通知
    現場端末->>工程管理: 工程着手登録
    工程管理-->>製造指示: ステータス更新（着手中）
    現場端末->>工程管理: 工程完了登録
    工程管理-->>製造指示: ステータス更新（完了）
```

### 品質検査〜出荷

```mermaid
sequenceDiagram
    participant 工程管理 as 工程管理
    participant 品質管理 as 品質管理
    participant 在庫管理 as 在庫管理
    participant 出荷管理 as 出荷管理
    participant ERP as ERP / 会計

    工程管理->>品質管理: 工程完了通知
    品質管理->>品質管理: 検査結果入力・合否判定
    品質管理->>在庫管理: 合格品 入庫登録
    品質管理->>出荷管理: 検査合格通知
    出荷管理->>在庫管理: 在庫引当・出庫
    出荷管理->>出荷管理: 出荷実績登録・納品書発行
    出荷管理->>ERP: 出荷実績・在庫残高（CSV日次）
```

## データ連携一覧

| 連携 | 方向 | 方式 | タイミング | データ |
|------|------|------|-----------|--------|
| 受注システム → 本システム | 受信 | Webhook（REST API） | リアルタイム | 受注番号・品目・数量・納期 |
| 本システム → ERP / 会計 | 送信 | CSVファイル | 日次（22:00バッチ） | 製造実績・在庫残高・出荷実績 |
| 調達システム → 本システム | 受信 | REST API | リアルタイム | 入荷予定・入荷実績 |
| 本システム → 調達システム | 送信 | REST API | アラート発生時 | 原材料発注依頼 |
| 現場端末 → バックエンド | 送信 | REST API | 都度 | 工程実績・検査結果 |
| バックエンド → 現場端末 | 送信 | SignalR（WebSocket） | リアルタイム | 製造指示・進捗通知・アラート |
